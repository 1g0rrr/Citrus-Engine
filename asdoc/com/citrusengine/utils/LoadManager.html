Classpublic class LoadManager
InheritanceLoadManager Object

The load manager keeps track of the loading status of Loader objects, returning an overall value for all graphics that are being loaded. This class is necessary when loading level graphics at runtime and finding out when all the graphics are finished loading. The LoadManager instance can be accessed via the state's CitrusView object. There is a LoadManager for each view state.

Method Detail
add()method
public function add(potentialLoader:*, recursionDepth:Number = 1):Boolean

The CitrusView calls this method on all graphics objects that it creates to monitor its load progress. It passes any object into the add() method, and it will recurse through it and search for any loaders on the object. If/when it finds a loader (or if it IS a loader), it will add it to the list of Loaders that it is monitoring. If you use Starling view, it can't be recursive, so we check if the StarlingArt's loader is defined.

Parameters

potentialLoader:* — The object that needs load monitoring.
 
recursionDepth:Number (default = 1) — How many child objects the add() method should recurse through before giving up searching for a Loader object.

Returns
Boolean — Whether or not it found a loader object.
