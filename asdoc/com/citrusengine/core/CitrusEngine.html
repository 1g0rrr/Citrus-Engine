CitrusEngine is the top-most class in the library. When you start your project, you should make your document class extend this class.

CitrusEngine is a singleton so that you can grab a reference to it anywhere, anytime. Don't abuse this power, but use it wisely. With it, you can quickly grab a reference to the manager classes such as current State, Input and SoundManager.

CitrusEngine can access to the Stage3D power thanks to the Starling Framework

consoleproperty 
console:Console  [read-only]

A reference to the console, so that you can add your own console commands. See the class documentation for more info. The console can be opened by pressing the tilde key (It looks like this: "~" right below the escape key). There is one console command built-in by default, but you can add more by using the addCommand() method.

To try it out, try using the "set" command to change a property on a CitrusObject. You can toggle Box2D's debug draw visibility like this "set Box2D visible false". If your Box2D CitrusObject instance is not named "Box2D", use the name you gave it instead.

stateproperty 
state:IState

A reference to the active game state. Acutally, that's not entirely true. If you've recently changed states and a tick hasn't occured yet, then this will reference your new state; this is because actual state-changes only happen pre-tick. That way you don't end up changing states in the middle of a state's tick, effectively fucking stuff up.
